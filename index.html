What lives behind this endpoint?

Node.js + Express service on Render, connected to API-Football plus internal models. This is the same engine that feeds the live demo UI and can be pointed at any data provider you already use.

1. Live stats & xG layer

Ingests live fixtures, shots, dangerous attacks and corners, then builds an approximate xG and pressure model for the current match state.

  • · Live fixture stream (API-Football or your own feed)
  • · xG approximation tuned for trading desks
  • · Shots on target, dangerous attacks, corners
  • · Normalised per match, league and provider

2. Odds & implied probabilities

Reads the market's opinion from your odds feed and converts it into clean, overround-free probabilities that can be tracked over time.

  • · 1X2 odds snapshot (home / draw / away)
  • · Overround-adjusted win probabilities
  • · Easy to compare vs your internal model
  • · Works with any odds provider

3. Edge & emotion ready

On top of stats and odds, BetSense can layer fan sentiment, pressure and momentum models (BetSense Emotion Index & BetSense LiveApe) without changing your feed.

  • · Emotion score per team (0–1)
  • · Momentum (home / away / neutral)
  • · Match-level “edge” explanation
  • · Ready to be plugged into UI or alerts

How the engine fits into your stack

BetSense Quantum Engine doesn't replace your sportsbook – it sits next to it. You point your live data and odds into the engine, and it returns a single, clean view of how the match is really behaving in this moment.

Step 1

Connect live data

Fixtures, events and odds from API-Football, Sportradar, Genius Sports or your own feed.

Step 2

Normalise & model

Standardised view of xG, volume, pressure and price for any league or provider.

Step 3

Expose clean APIs

Simple endpoints for live stats, odds and probabilities – plus emotion where enabled.

Step 4

Plug into UI / tools

Use our UI or your own dashboards, alerts and risk tools. The engine stays the same.

API overview (current demo endpoints)

In this demo environment, the engine exposes three core read-only endpoints. In production, these can be namespaced, authenticated and extended with client-specific logic.

GET /api/stats Live match pulse

Returns xG (home), shots on target, dangerous attacks and corners for the current match in focus. 

{
  "source": "demo | live",
  "stats": {
    "xg": 1.32,
    "shots_on_target": 4,
    "dangerous_attacks": 28,
    "corners": 5
  }
}
GET /api/live-odds Market snapshot

Reads 1X2 odds from your feed and returns a clean “home / draw / away” structure for downstream models. 

{
  "source": "demo | live",
  "odds": {
    "home_win": 1.85,
    "draw": 3.20,
    "away_win": 4.50
  }
}
GET /api/probabilities Implied probabilities

Converts odds into normalised probabilities, ready for overlays such as edge, emotion and fair price. 

{
  "source": "demo | live",
  "probabilities": {
    "home_probability": 0.54,
    "draw_probability": 0.24,
    "away_probability": 0.22
  }
}

BetSense is an analytics engine only. It does not process or accept bets and is not a sportsbook.
